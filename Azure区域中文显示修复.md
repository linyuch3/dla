# Azure区域中文显示修复

## 🔧 问题描述
在之前的Azure功能修复中，动态获取区域功能导致区域名称显示为英文（如"East US", "West US 2"等），而不是原来的中文名称。

## ✅ 修复内容

### 1. 添加区域中英文映射表
创建了完整的Azure区域中英文对照表，包含：
- **美洲区域**：美国东部、美国西部、加拿大等
- **欧洲区域**：西欧、北欧、英国、法国、德国等  
- **亚太区域**：东亚、东南亚、日本、韩国、澳大利亚等
- **其他区域**：巴西、南非、印度、阿联酋等

### 2. 修改动态区域获取逻辑
- 在`getAvailableRegions()`方法中使用映射表
- 优先显示中文名称，如果映射表中没有则显示英文名称
- 保持动态获取的优势，同时确保中文显示

### 3. 完整的区域映射
```typescript
private readonly regionNameMap: Record<string, string> = {
  'eastus': '美国东部 弗吉尼亚',
  'westus2': '美国西部2 华盛顿',
  'westeurope': '西欧 荷兰',
  'southeastasia': '东南亚 新加坡',
  // ... 更多区域映射
};
```

## 🎯 修复效果
- ✅ Azure区域选择下拉框现在显示中文名称
- ✅ 保持动态获取可用区域的功能
- ✅ 如果新区域暂无中文映射，会显示英文名称作为回退
- ✅ 不影响其他云服务商的区域显示
